# data-ops

Shared DB layer consumed by user-application and data-service.

## Structure

```
src/
├── {domain}/          # Domain barrels (client/, health/, etc)
│   ├── table.ts       # Drizzle table definition
│   ├── schema.ts      # Zod schemas + types
│   ├── queries.ts     # DB operations
│   └── index.ts       # Barrel re-export
├── drizzle/
│   ├── schema.ts      # Main application tables
│   ├── auth-schema.ts # Auto-generated by Better Auth, don't edit
│   ├── relations.ts   # Cross-domain joins config
│   └── migrations/    # Per-env dirs: dev/, staging/, production/
├── database/
│   ├── setup.ts       # getDb() connection
│   └── seed/          # Data seeding utilities
└── auth/
    ├── setup.ts       # Auth config (providers, plugins)
    └── server.ts      # Auth server instance
```

## Consumer imports

```ts
import { getClient, ClientSchema, type Client } from "@repo/data-ops/client"
import { checkDatabase, type LivenessResponse } from "@repo/data-ops/health"
```

## New domain workflow

1. Create `src/{domain}/table.ts` with Drizzle table
2. Add relations to `src/drizzle/relations.ts` if needed
3. `pnpm run drizzle:dev:generate` + `pnpm run drizzle:dev:migrate`
4. Create `src/{domain}/schema.ts` (Zod schemas)
5. Create `src/{domain}/queries.ts`
6. Create `src/{domain}/index.ts` (barrel)
7. Add export to `package.json`
8. `pnpm run build`

## Don't

- Edit `auth-schema.ts` directly (auto-generated by Better Auth)
- Forget to rebuild after changes (`pnpm run build`)
